# Rebuttal

Use this file when the user has reviewer comments, wants to prepare a rebuttal, or needs to decide which additional experiments are worth doing under rebuttal time pressure.

## Goal

A good rebuttal does three things:

1. classify the reviewer concerns correctly
2. answer each concern with the strongest available evidence
3. avoid overclaiming what cannot be fixed within the rebuttal window

## Triage

Group reviewer comments into:

- misunderstanding
- missing evidence
- weak writing or framing
- weak baseline or comparison
- methodological concern
- scope limitation the paper cannot fully resolve now

This classification matters because the response style should differ.

## Response Structure

For each reviewer point, write:

- reviewer concern
- your interpretation of the real issue
- direct answer in 1-2 sentences
- supporting evidence
- additional experiment or clarification
- scope limit, if unresolved

Do not bury the answer inside background exposition.

## Prioritization

When time is limited, prioritize:

1. comments that attack the main claim
2. comments that multiple reviewers share
3. comments that can be answered with one strong clarification or experiment
4. comments where the current wording caused the problem

Do not spend the whole rebuttal budget on minor stylistic complaints if a core validity concern remains open.

## Evidence Rules

- If an experiment is already available, cite it directly and clearly.
- If a new experiment is feasible, state the setup compactly and report the outcome without drama.
- If the concern cannot be fully resolved now, acknowledge the limit precisely and narrow the claim if needed.
- Never bluff confidence where evidence is missing.

## Tone

The tone should be:

- direct
- respectful
- non-defensive
- evidence-led

Do not write as if the reviewer is an opponent. Do not write as if the reviewer is obviously wrong. Translate emotion into structure.

## Typical CV/ML Rebuttal Patterns

Common concerns include:

- weak or missing baseline
- dataset bias or weak evaluation protocol
- ablation not isolating the claimed component
- gain explained by scale, compute, or extra supervision
- poor qualitative examples despite metric gains
- unclear novelty relative to a close prior paper

Treat these as high-value rebuttal targets because they often affect the decision more than cosmetic issues.

## Rebuttal Decision Rule

Before recommending more experiments, ask:

- does this experiment directly answer a reviewer concern
- can it be completed reliably within the deadline
- will it materially strengthen the main claim

If not, prefer a tighter clarification over a rushed weak experiment.
